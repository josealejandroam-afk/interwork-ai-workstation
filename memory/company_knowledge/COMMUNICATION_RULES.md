# Communication Rules — InterWork

## Tone

- Concise, professional, corporate
- Direct and factual — no filler phrases
- Friendly but not casual
- Use complete sentences in client-facing messages

## Language Rules

| Rule | Detail |
|---|---|
| Do not use "lock in" | Avoid this phrase in all drafts |
| Use "our team" or "InterWork" | Not "my guys", "the crew", "the movers" |
| Do not name vendors | Never include vendor company name or warehouse address in client-facing communications |
| Do not reference internal pricing | Never share margins, vendor rates, or cost breakdowns with clients |
| Include PM full name and phone in client notices | When dispatching or confirming to a client, include the assigned field PM's name and number |
| Do not invent project numbers | Only use project numbers that exist in Supabase or the project index |
| Do not invent contacts | Only use names and phone numbers already confirmed in the project card |
| Ask when unclear | If a key detail is missing (who to address, which phone, which date), ask Alejandro rather than guessing |

## Client Email Format

- Subject: `[Client name] - [Short description] - [Date or action]`
- Open with the job dates and purpose
- State the field PM name and phone number
- Include any specific requests or open items clearly
- Close with contact info (Alejandro Acosta, InterWork, phone)
- Do not include: vendor names, warehouse, internal cost data, ChatGPT references

## Teams Message Format

- Short and direct — one screen max
- State: project number, client, dates, location, inventory (if relevant)
- State what the PM needs to do or confirm
- Include the client contact name if the PM may need to reach them
- Sign off with name if needed

## FastField Notes Format

- Project number, client, date, PM name
- Origin address (full) and/or destination address (full)
- Scope: bullet list of what to do
- Special notes: anything requiring attention
- Do not include internal margin data

## Approval Required Before Sending

No email or Teams message is sent without Alejandro saying "send it."
No Supabase write is executed without Alejandro saying "approve" or "apply."
Drafts are always free — sending is never automatic.
