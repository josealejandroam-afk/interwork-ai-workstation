# InterWork Information Flow

_Last updated: 2026-06-30_

---

## Overview

This document maps how work information enters the InterWork AI system, moves through it, and produces outputs. It is the operational companion to `FINAL_AI_INFORMATION_ARCHITECTURE.md`.

---

## Inbound Signals

| Signal | Source | How It Enters | Where It Lands |
|---|---|---|---|
| New project request | Client email / SOW | Alejandro pastes to Claude | Supabase `projects` table |
| Project update | Client Teams message | Graph API (read) or paste | Open loop → project card |
| Field completion | FastField form submit | Webhook → Make.com | `fastfield_webhook_events` |
| Meeting notes | Read AI | MCP read | `memory/inbox/` or project card |
| Vendor confirmation | Email/Teams | Paste to Claude | Project card update proposal |
| PM status update | Teams DM | Graph API or paste | Project card update proposal |

---

## Outbound Outputs

| Output | Destination | How It Leaves | Approval Required |
|---|---|---|---|
| Client confirmation | Email / Teams | Drafted by Claude, sent by Alejandro | Yes |
| Vendor dispatch | Email / Teams | Drafted by Claude, sent by Alejandro | Yes |
| FastField form | Field PM via email | Triggered by Alejandro | Yes |
| Project status update | Supabase | Proposed by Claude, applied by Alejandro | Yes |
| Open loop close | Supabase | Proposed, applied with approval | Yes |
| Client knowledge pack | Claude Chat Project | Regenerated by Claude, uploaded by Alejandro | Upload step manual |

## Signal Quality Tiers

| Tier | Signal | Confidence | Action |
|---|---|---|---|
| High | FastField webhook + WC report | Very high | Can propose close with confidence |
| Medium | Teams message confirming done | Medium | Needs cross-check with FastField |
| Low | Scheduled date passed, no signals | Low | Flag for follow-up, do not close |
| None | No data beyond project entry | None | Flag as attention item |

## Information Routing Rules

### Before answering any project question:
1. Check `memory/company_knowledge/START_HERE.md`
2. Check `memory/clients/<client>/CLIENT_CONTEXT.md`
3. Check `memory/clients/<client>/projects/<slug>/PROJECT_CARD.md`
4. If card missing: ask Alejandro to paste it — never guess

### Before drafting any communication:
1. Check `memory/procedures/interwork_communication_rules.md`
2. Confirm recipient is in `memory/references/interwork_people_map.md`
3. Confirm project details from project card — never from memory alone
4. Draft only — never send

### Before proposing any Supabase write:
1. Show the proposed change as a table: field → current value → proposed value
2. State confidence level and source of evidence
3. Wait for explicit approval — "yes" is enough; never auto-apply

## Blocked Flows (Do Not Route)

| Flow | Why Blocked |
|---|---|
| Raw emails to repo | PII / confidentiality |
| Outlook/M365 auto-read | Auth complexity, not set up |
| Claude → Supabase auto-write | Requires human approval |
| Secrets / tokens in any file | Security policy |
| Claude Chat → GitHub push | Chat has no repo access |
